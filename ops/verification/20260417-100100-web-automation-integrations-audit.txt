  in .yaml — no central config map.

5. Better replacements or package suggestions
------------------------------------------------------------------------------
- Playwright → stick with it. It's the de facto standard.
  browser-use (an LLM-driven Playwright wrapper) is referenced in
  AGENTS.md; fine for autonomous tasks but not an upgrade needed.
- Selenium / Puppeteer: not worth moving to.
- For D-Tools specifically: prefer the REST client whenever
  possible and reserve Playwright for endpoints with no API.
- Consider a tiny `ops/INTEGRATIONS.md` matrix listing every
  external service, its auth mechanism, env var name, and the
  code path that calls it. Cheap orientation win.
