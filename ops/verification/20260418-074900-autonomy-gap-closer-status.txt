SECTION 7 — Limitations / TODOs
================================================================

* No dedicated staging host. Dry-run is the staging lane today.
  Add an explicit note in any high-risk plan.
* Handlers other than run_cline_prompt / run_cline_campaign do not
  honor dry_run internally. If a run_script task writes to disk
  or hits external systems, either make the script honor DRY_RUN/
  --dry-run, or skip the dry-run lane and rely on the approval-
  token gate.
* The gate does not inspect .approval file contents — presence
  plus commit history is the whole audit trail. Include any
  justification metadata in the commit message.
* AUTO_APPROVE_IDS.txt is intentionally empty; add task_ids only
  after explicit human sign-off on a recurring high-risk op.
