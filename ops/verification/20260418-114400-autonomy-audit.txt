3. Pros (what's objectively strong)
------------------------------------------------------------------------------
- 20 tasks have landed in completed/ over the life of the runner, zero
  failed, zero rejected. When the runner is up, it works.
- Approval-gate (task_runner_gates.py) is cleanly pluggable — if the
  module is absent, the runner warns once and treats every task as
  low-risk (graceful degradation).
- Dry-run lane: tasks with `dry_run: true` bypass the approval gate
  entirely and propagate the flag into the handler payload. Cline-
  prompt and cline-campaign launchers honor it end-to-end.

  remote agent can see pending/stale/oldest without touching the
  runner.
- Audit-index CLI (`task_audit_index.py <task_id>`) resolves the
  entire artifact chain in one call — task JSON → prompt files →
  verification artifacts → git commits. This is the forensic tool
  that didn't exist on 4/17.
- Smoke test (`test_task_runner_gates.py`) runs in ~0.1s and covers
  8 meaningful gate decisions including path traversal in
  approval_token.
- Preflight is now idempotent + silent on no-op ticks (4/18 fix),
  which removes the last source of commit spam.

5. Better replacements or package suggestions
------------------------------------------------------------------------------
Re-validated the April 17 verdict:
- Celery / RQ / Dramatiq: overkill, brokers become new failure modes.
- Temporal / Airflow: much harder to audit than a JSON file in git.
- GitHub Actions: violates the "everything on Bob" rule.
- Hashicorp Vault: over-engineering for a 2-key, 1-agent system.

What would materially improve autonomy — without swapping the stack:
1. depends_on + priority metadata (small, high leverage).
2. Auto-retry ssh_and_run on transient exit codes (e.g. 255) with
   exponential backoff, bounded at 3 attempts.
3. Externalize ALLOWED_TASK_TYPES to ops/work_queue/ALLOWLIST.json
   so adding a type is a config commit, not a Python commit.

Net verdict: v1 shape is still correct. V2 is not warranted. The
gaps are additive features, not architectural.
