## What was built

Four files under integrations/x_intake/reply_actions/:

  parser.py       — pure parse_reply(raw, valid_slots) → ParsedReply
                    No I/O, no side effects, fully unit-testable.
                    Handles: "1", "reply 1", "reply1", "r1", "R1", trailing text.
                    Ambiguity: multiple valid-slot digits in one reply → status="ambiguous".
                    Unknown/out-of-range → status="unrecognized".

  action_store.py — ActionStore class backed by SQLite at
                    /data/x_intake/reply_actions.db (existing Docker volume).
                    API: create() → action_id (12-char hex, 6 bytes via secrets.token_hex)
                         lookup(action_id) → ActionContext | None
                         mark_used(action_id, slot) → bool (first-writer-wins dedupe)
                         prune() → int (deletes expired rows)
                    NOT wired to any handler — Phase 4 adds execution.

  formatter.py    — format_card(summary, action_id, expiry_seconds, slots) → str
                    Reads action catalog from config/reply_actions.schema.json at import.
                    Produces: {summary}\n\n──────────\n{options_line}\nID:{id} · exp {Nh}
                    strip_options_block() reverses the append for re-formatting.

  __init__.py     — Re-exports: parse_reply, ParsedReply, ActionStore, ActionContext,
                    format_card, strip_options_block.
