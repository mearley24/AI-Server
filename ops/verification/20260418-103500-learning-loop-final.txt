Scope of this pass
------------------
Added the missing self-learning layer to the autonomous execution pipeline:

  observe  ->  extract lessons  ->  promote to guardrails  ->  teach Matt

Everything is repo-native, stdlib-only Python, no new services or APIs.
No existing task-runner / preflight / gate code was modified. The
learning loop reads the stream of artifacts that stage already produces
and turns it into durable knowledge.

How to run the loop manually
----------------------------
  python3 ops/learning_miner.py  --days 7  --update
  python3 ops/learning_digest.py --days 7  --write

Both scripts have --help, accept --out PATH, and are fully idempotent.
The digest always writes to ops/verification/ so the output is committed
like any other verification artifact.

Scheduling
----------
No scheduler was wired in this pass. Two suggested entry points:

  (a) Queue a weekly run_script task under ops/work_queue/pending/:
      shell="python3 ops/learning_miner.py --days 7 --update
             && python3 ops/learning_digest.py --days 7 --write"
      task_type=run_script, risk_tier=low, no approval needed.
  (b) A daily miner-only tick, plus the weekly digest above. Keeps
      last_seen_at current without flooding the verification dir.

Both options are additive; no new launchd plist needed because the
existing com.symphony.task-runner plist already dispatches run_script.

Top 3 next improvements
-----------------------
1. Tighter mining heuristics. The first run picked up a few summaries
   that are really table fragments (column counts from a brain.db
   category report). A simple "skip if body looks tabular" filter and a
   per-file ceiling on candidates will cut false positives.

2. Feed-the-backlog loop. For every lesson whose status stays "new" more
   than 7 days and has a meaningful impact_hint, auto-queue a low-risk
   verify_dump task asking the next agent to either resolve, promote,
   or mark superseded. Closes the observe->learn->act cycle entirely
   inside the task runner.

3. Staging-mode miner for more aggressive learning. Current miner is
   conservative (heading-driven). A staging-mode variant could also
   cluster similar free-text paragraphs and propose higher-level lesson
   candidates for human review. Kept out of this pass deliberately to
   avoid over-engineering a v1.
