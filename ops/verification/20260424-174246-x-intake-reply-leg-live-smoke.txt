   Fix: decode bytes before json.loads.

   which is not installed in the x-intake container. Exception was caught
   silently (str(exc) == ""). Fix: call Cortex POST /api/bluebubbles/send
   via httpx instead (CORTEX_URL env var).

3. Action context schema: create() was called without slot_handler_map key,
   causing reply_unknown_handler. Context must include:
   {"slot_handler_map": {"1": "cortex_remember"}, "thread_guid": "...", ...}

4. routing JSON: +18609171850 was added with leading + but BlueBubbles
   may send handle address without the + prefix. Fix: added both forms
   (with and without +) and added iMessage;-;+18609171850 to allowed_chat_guids.
