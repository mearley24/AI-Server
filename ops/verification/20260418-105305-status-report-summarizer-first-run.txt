The summarizer internally:
  1. Reads STATUS_REPORT.md from the repo root.
  2. Parses H2 sections (`Now`, `Next`, `Later`, `Done`, `Reference: ...`).
  3. Classifies each top-level bullet as open / done / followup / needs_matt.
  4. Writes the digest to ops/verification/<stamp>-status-report-summary.md.
  5. Persists a snapshot to data/status_report_summarizer/last_snapshot.json
     (gitignored via the existing `data/**/*.json` rule).
