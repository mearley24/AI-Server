------------------------------------------------------------------------------
Safer replacements where clearly better
------------------------------------------------------------------------------
Cortex + SQLite remains the best stack today. Concrete alternatives
and their verdicts:

- **PostgreSQL + pgvector** — real rewrite, only worth it if we
  cross ~500 k brain.db rows or start running concurrent writers.
  Not today.
- **litestream** — single-node continuous SQLite backup to S3/B2.
  Low intrusion. Best candidate for "scheduled backup" work.
- **LiteFS** — streaming replication with FUSE. Overkill for
  single host; reconsider if a second node joins.
- **sqlite-vss** — load as SQLite extension on brain.db, keep the
  file. Plan for when brain.db >100 k rows. Embedding cost at
  current write rate ≈ $0.12/day.
- **Mistakes registry over Cortex** — not a replacement, a
  *layer*. This is the highest-leverage upgrade: promote
  ops/verification/*-blocker-*.txt to brain.db/cortex.db rows so
  recurring failures can be detected automatically.
