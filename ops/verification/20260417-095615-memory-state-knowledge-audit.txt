3. Pros
------------------------------------------------------------------------------
- Cortex is the single sink — no "which DB owns this fact" debate.
  37k+ memories, 20+ categories, growing ~1/min from wired services.
- Schema supports importance, confidence, TTL — the three fields
  you need for intelligent pruning.
- Dashboard surface exposes both raw lists and filtered queries.
- Apple Notes ingestion is committed code (integrations/apple_notes/
  + multiple tools/notes_*.py), not a bespoke shell script.
- follow_ups.db consolidation (2026-04-17) resolved a three-way
  split into a single canonical file.
- Schema migrations live in cortex/migrate.py (not ad-hoc SQL).
- Daily digests are automatic and human-readable.

4. Cons / weaknesses
------------------------------------------------------------------------------
- DATABASE_SCHEMA.md (89 lines) covers only jobs.db +
  decision_journal.db. Not authoritative for cortex/brain.db,
  knowledge_base.db, openclaw_memory.db, payments.db,
  cost_tracker.db, price_monitor.db, or emails.db. A future agent
  has to read SQL to learn the shape.
- No embedding / vector index. At 37k rows SQL lookup is fine;
  at 1M it won't be. No migration plan.
- Three categories (trading_strategy, risk_management, follow_up)
  own 86% of brain.db. No per-category retention policy exists;
  PRUNE_CONFIDENCE_THRESHOLD and PRUNE_STALE_DAYS are global.
- No top-level manifest for `knowledge/` — agents discover files
  by grep/ls, which means tools hardcode paths.
- No scheduled backup for brain.db, decision_journal.db,
  follow_ups.db. Only ad-hoc `scripts/backup-data.sh` + one tarball
  from April 5 (12 days ago).
- price_monitor.db hasn't updated since April 10 — possible stale
  pipeline. Needs investigation.
- No cost tracking display for the 17,062 trading_strategy memories
  (how much did researching them cost across Perplexity/OpenAI?).
- Apple Notes ingestion leans on Notes.app being running — no
  degraded mode when the host Notes.app is absent.

------------------------------------------------------------------------------
5. Better replacements or package suggestions
------------------------------------------------------------------------------
- Chroma / LanceDB / sqlite-vss: local vector DB for future
  embedding search. Worth planning once brain.db passes ~100k rows,
  not today.
- litestream: streaming SQLite replication to a remote (e.g. B2,
  S3). Would give near-continuous backup. Strong fit for brain.db.
- DuckDB: faster analytical queries over SQLite files. Optional
  overlay for dashboard aggregations.
- jq + CLI tooling for knowledge/ contents: add a
  `knowledge/INDEX.json` manifest keyed by relative path →
  description, so tools can discover content without hardcoding.

Net: no urgent swaps. A few targeted upgrades (litestream, a
knowledge manifest) could be added when time allows.
