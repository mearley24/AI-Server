How to run the summarizer manually
----------------------------------
```
cd /Users/bob/AI-Server
python3 ops/status_report_summarizer.py --write
```

Outputs:
  - ops/verification/YYYYMMDD-HHMMSS-status-report-summary.md
  - data/status_report_summarizer/last_snapshot.json  (gitignored)

Flags:
  --write           also write the digest to ops/verification/
  --out PATH        explicit output path (overrides the default name)
  --print           force stdout (default when --write is absent)
  --no-snapshot     do not update the diff baseline (useful for
                    one-off previews)
  --status-report PATH   alternate STATUS_REPORT.md location

Wiring into the task runner
---------------------------
A shell wrapper lives at
`ops/task_runner/scripts/status-report-summary.sh`. Queue a task JSON
under `ops/work_queue/pending/` with this payload:

```
{
  "task_type": "run_script",
  "payload": {
    "script": "status-report-summary",
    "args": []
  },
  ...task_id, created_by, created_at, signature...
}
```

On the next runner tick the wrapper executes, the summarizer writes a
new digest, and the runner commits the result into the verification
stream as usual. Low-risk; no approval token required.

Scheduling recommendation (no launchd installed in this pass):

  - For hands-free operation, add a `StartCalendarInterval`-based

    script is idempotent and cheap (~60ms wall-clock), so more

Tagging conventions
-------------------
The summarizer recognises two bullet-level tags inside STATUS_REPORT.md
top-level sections (`Now` / `Next` / `Later`):

  `- [FOLLOWUP] description...`    grouped under "Follow-ups"
  `- [NEEDS_MATT] description...`  grouped under "Needs Matt"
  `- ~~title~~ ✅ ...`             treated as done; retained as history

Legacy prose markers continue to trigger the same classification:
  - `[Matt]`, `Needs Matt`, `Awaiting Matt`, `Requires Matt`,
    `pending Matt`, `Matt action`
  - `Fund ... wallet`
  - `KRAKEN_SECRET`
  - `Requires approval` / `Explicit approval`
  - `Follow-up`, `Follow up`
  - `Recommended next action`, `Remaining follow-up`

Section defaults:
  - Items in `Later` default to follow-up unless explicitly done.
  - Items in `Now` default to "surface prominently" (tagged as
    follow-up for visibility) when they lack an explicit
    `[NEEDS_MATT]` marker.
