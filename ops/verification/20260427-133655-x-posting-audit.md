## 5. Why Posting Has Been Dormant Since 2026-04-16

The scheduler (`com.symphony.x-autoposter`) has been running 7×/day since 2026-04-16 but **all content generation attempts fail** with:

```
ModuleNotFoundError: No module named 'requests'
```

Root cause: plist uses `/usr/bin/python3` (macOS system Python / CLT), which does not have `requests` installed. Homebrew Python (`/opt/homebrew/bin/python3`) does. Every scheduler run since the first post has ended with "No posts to send."

The scheduler itself has been running continuously — it is **not disabled**. It would immediately resume posting if the Python interpreter path was corrected.
