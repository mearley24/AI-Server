## Usage Guard Proof

`usage.py:check_limit()` reads `x_api_usage` table, sums `request_count` for today UTC. If sum >= `X_DAILY_READ_LIMIT` (default 100), `run_intake()` returns `{"status": "limit_reached"}` before making any API call. Verified by `TestUsageLimit::test_limit_enforced`.

## No X Write Actions Exist

- `XReadOnlyClient` monkey-patches all write methods with `_write_blocked()` which raises `RuntimeError("...read-only")` immediately
- `_WRITE_METHODS` frozenset covers: create_tweet, delete_tweet, like_tweet, unlike_tweet, retweet, unretweet, follow_user, unfollow_user, create_direct_message, create_list, delete_list, add_list_member, remove_list_member, mute_user, unmute_user, block_user, unblock_user, bookmark_tweet, remove_bookmark
- No write endpoints in cortex — only GET /status, GET /items, POST /dry-run (preview only)
- `should_auto_run: false` hardcoded in all response paths
