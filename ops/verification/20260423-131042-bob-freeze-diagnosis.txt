## Phase 7 — Synthesis + fix decision

Primary root cause (high confidence, static evidence):
  scripts/task_runner.py runs git pull/push/status/commit via subprocess.run WITHOUT timeout=.
  Combined with fcntl.flock single-instance locking, one stalled git call wedges all
  subsequent launchd ticks. Visible symptom matches "Bob freezes up."

Secondary root cause:
  scripts/bob-watchdog.sh docker_healthy probes docker info / docker ps without --timeout.
  In Docker Desktop zombie-daemon state (documented in-repo on 2026-04-21) these can hang.

Not-a-cause (ruled out via static evidence):
  - cortex loops are well-structured; all httpx/redis/subprocess calls carry explicit timeouts.
  - x_intake listener + watchdog use asyncio.sleep backoff; httpx all timed.
  - bluebubbles uses configured self.timeout on all httpx calls.

Safe-low-risk-fix gate: PASS for the proposed Phase 1 change (<=40 LOC, one file primary + one watchdog, no launchd/docker/redis/auth/secrets touched, adds bounded timeouts only, smoke test easy, no manual restart needed — launchd re-reads the file each 120s tick).
