4. Cons / weaknesses
------------------------------------------------------------------------------
- No consolidated TRADING_STRATEGIES.md doc — 27 strategy files,
  each with its own params. A future trader/agent has no map.
  Analogous to the openclaw MODULE_MAP gap.
- Lesson #17 (sell haircut rounding) still UNKNOWN.
- Kalshi still in demo mode (STATUS_REPORT Later).
- No automated alert on redeemer failure or bankroll <$10.
  (See MESSAGING_ALERTS.md — TODO rows added.)
- paper_ledger score isn't surfaced on the dashboard trading tile
  (only live P&L shows).
- No strategy ablation logs — hard to tell which strategy cost or
  saved money on which day.
- Bot is VPN-only. If the VPN container restarts mid-tick a trade
  can be torn down silently. No explicit "missed tick" counter in
  /status.
- wallet_rolling_redis + wallet_scoring + whale_scanner produce
  a lot of Redis chatter. No per-channel TTL / size metric.
- data/polymarket/ is gitignored runtime; no backup path for
  trades.csv, although it is replayable from on-chain events.
